# Quickstart

## When to use
Use this skill when:
- the repo already has many pages, project entries, and archives
- the same content keeps being rewritten across multiple pages
- a team wants personal and collective views to coexist
- the repo needs to feed agent workflows, websites, and media kits

## 10-minute version
1. scan the repo
2. mark candidates as:
   - work
   - node
   - person
   - collective
   - view-only
3. assign owner/context
4. export starter database
5. draft note plan
6. validate the distinction between objects and views

## First wave recommendation
Only build:
- 3–5 works
- 5–10 nodes
- 3–10 people
- 1 collective
- 10+ relations

Do not start with everything.

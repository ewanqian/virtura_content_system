# Implementation Plan

## Stage 1 — Personal-site prototype
Use `portfolio` as the first running instance.

Goals:
- structure works and nodes
- generate data for Selected Works and Image Wall
- stabilize link and image rules
- begin content validation

## Stage 2 — Shared protocol repo
Create `VIRTURA-Content`.

Goals:
- move content model out of a single site
- define schemas
- define site profiles
- define generated outputs for multiple public layers

## Stage 3 — Multi-site consumption
Connect generated data to:

- portfolio
- collective site
- SpacePort
- Newsroom
- viewer pages

## Stage 4 — Skill-based operations
Add scripts and skills for:

- ingestion
- validation
- generation
- sync
- route checking
- image indexing

## Immediate priority

Do not start with a heavy admin dashboard.

Start with:
- schema
- file structure
- relation rules
- generation scripts
- link validation

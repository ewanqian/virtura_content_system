🚀 快速开始

3分钟完成安装配置，开始使用Virtura Node Weaver MCP服务

📋 环境要求

确保您的系统满足以下要求

🐍

Python 3.8+

用于运行CLI工具和数据处理脚本

python3 --version
# 输出: Python 3.8.0 或更高版本
📦

Node.js 16+

仅MCP服务需要Node.js环境

node --version
# 输出: v16.0.0 或更高版本

📥 安装步骤

  1. 1. 克隆仓库

    首先，克隆项目仓库到本地

    git clone https://github.com/ewanqian/virtura_content_system.git
cd virtura_content_system

  2. 2. 安装Python依赖

    安装CLI工具和数据处理脚本的依赖包

    pip install -r cli/requirements.txt
pip install -r scripts/requirements.txt

  3. 3. 安装MCP服务依赖

    进入mcp_server目录并安装Node.js依赖

    cd mcp_server
npm install
cd ..

  4. 4. 配置MCP服务

    复制示例配置文件并进行必要的修改

    cp config/mcp_config.example.yaml config/mcp_config.yaml

    编辑 config/mcp_config.yaml 文件，配置以下选项：

    • 服务器配置：端口和主机地址
    • 内容路径：内容存储目录
    • 安全设置：API密钥和访问控制
    • 日志选项：日志级别和输出位置

  5. 5. 初始化内容系统

    运行初始化脚本，创建必要的目录结构

    python3 scripts/factory_ingest.py init

  6. 6. 启动MCP服务

    使用标准脚本启动服务（默认作为stdio服务器）

    ./start_mcp_server.sh

    或者，您可以使用Node.js直接启动：

    cd mcp_server
npm start

  7. 7. 验证服务

    启动后，您可以使用以下方法验证服务是否正常运行：

    cd mcp_server
npm test

💻 配置Claude Desktop

让Claude Desktop能够访问您的MCP服务

  1. 1. 定位配置文件

    在您的系统上找到Claude Desktop的配置文件：

    • macOS：~/Library/Application Support/Claude/config.json
    • Windows：%APPDATA%/Claude/config.json
    • Linux：~/.config/Claude/config.json

  2. 2. 编辑配置文件

    在配置文件中添加MCP服务器配置：

    {
  "mcpServers": {
    "virtura-content": {
      "command": "node",
      "args": ["/path/to/virtura_content_system/mcp_server/src/index.js"]
    }
  }
}

  3. 3. 验证连接

    重启Claude Desktop后，您应该能在工具列表中看到Virtura Content System的工具。

    您可以通过以下方式验证：

    // 在Claude Desktop中测试
content_system_list_types()

🎯 基本使用

完成安装后，您可以通过以下方式使用Virtura Node Weaver MCP

💬

Claude Desktop

在Claude Desktop的对话中直接使用工具：

// 搜索内容
content_system_search({
  "query": "AI",
  "type": "work"
})

// 创建新作品
content_system_create({
  "type": "work",
  "fields": {
    "title": "AI生成艺术",
    "description": "探索AI在艺术创作中的应用",
    "tags": ["AI", "艺术", "创新"],
    "date": "2026"
  }
})
⚙️

命令行工具

使用CLI工具管理内容：

./weaver list --type work
# 列出所有作品

./weaver create --type node --title "AI生成工作流"
# 创建新节点

./weaver link --object1 work:drop-flow --object2 node:ai-generation-workflow --relation uses_method
# 建立关联

🏭 跑通第一个流程

使用投料工厂批量导入和处理内容

  1. 1. 准备素材

    将您要导入的素材（图片、文档、聊天记录等）放入incoming目录

    ls -la incoming/
# 显示目录内容
# image1.jpg
# document.md
# chat_history.json

  2. 2. 批量导入

    使用投料工厂批量导入素材，生成待审核清单

    python3 scripts/factory_ingest.py import --dir incoming/
# 输出处理结果

  3. 3. 查看待审核项目

    列出需要审核的待确认项目

    python3 scripts/factory_ingest.py review --list
# 输出待审核项目列表

  4. 4. 审核确认

    使用自然语言指令审核确认项目

    python3 scripts/factory_ingest.py review --confirm "批准所有"
# 审核通过，生成标准对象

  5. 5. 构建数据

    运行完整流水线，生成可视化数据

    python3 scripts/build_pipeline.py
# 构建数据并生成viewer所需的文件

  6. 6. 查看可视化

    启动本地服务查看可视化界面

    python3 -m http.server 8080
# 打开浏览器访问: http://localhost:8080/viewer/

❓ 常见问题

服务启动失败

问题：npm start命令失败
解决：检查Node.js版本是否符合要求，或者重新安装依赖 

cd mcp_server
rm -rf node_modules package-lock.json
npm install

Claude Desktop无法连接

问题：配置后Claude Desktop看不到服务
解决：检查config.json路径是否正确，或尝试重启Claude Desktop

CLI命令不识别

问题：./weaver: 没有那个文件或目录
解决：检查weaver脚本的执行权限 

chmod +x weaver
# 或使用Python直接运行
python3 cli/weaver.py

数据导入失败

问题：factory_ingest.py import命令失败
解决：检查incoming目录权限和文件格式

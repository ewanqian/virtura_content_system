# Content Protocol Station

## What this station is for

Content Protocol Station is the public-facing station that explains how VIRTURA organizes works, nodes, assets, writings, venues, specs, and relations across multiple repositories and sites.

It is a station about content infrastructure.

This station does not function as the actual content source repository itself.
Instead, it introduces the protocol, the object model, the workflows, and the relationship between VIRTURA's public layers and internal content systems.

## Why this station exists

VIRTURA is not building only one website.

Its public ecosystem already includes multiple layers, such as:

- a collective-facing entry
- a public SpacePort
- a newsroom / commentary layer
- artist-facing pages
- scene / viewer tools
- archive and exhibition pathways

As this ecosystem grows, the key problem is no longer page design alone.
The key problem becomes: how should works, people, events, images, research texts, and technical specs be structured so that multiple sites can read from the same source?

This station exists to answer that question.

## What it does

This station helps explain:

- what the VIRTURA distributed content protocol is
- what object types it uses
- how content is routed into different site profiles
- how skills and agents can operate on structured content
- how content infrastructure supports archive, exhibition, publication, and viewer systems

## What it does not do

- It is not the actual CMS backend.
- It is not a page builder.
- It is not a substitute for SpacePort, Collective, or Newsroom.
- It is not a single product layer.

## Current focus

The current focus of this station is to define and explain:

1. the core object model
2. the relationship between content and public site layers
3. a file-based workflow that can be maintained with GitHub, agents, and scripts
4. how the same content object may appear in multiple public contexts

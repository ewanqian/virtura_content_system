VIRTURA-
Content

这不是一个传统网站后台，也不是一个单站点 CMS。它是一套面向作品、节点、图片、文章、场地、技术规格与关系映射的 file-based 内容协议，用来支撑 VIRTURA 的个人网站、团队网站、SpacePort、Newsroom、viewer 与后续多艺术家页面。

它服务的不是某一个首页，而是一整套去中心化内容网络：同一份内容可以被 portfolio 读成代表作品，被 Collective 读成协作节点，被 SpacePort 读成站点入口，被 Newsroom 读成评论对象，也可以被 viewer 读成技术与场地规格。

Overview

它是什么，不是什么

VIRTURA-Content 的核心任务，是把内容从页面中抽离出来，转成可被多站点、脚本、agent 和 skills 共同理解与操作的结构化对象。它不是一个给非技术用户点点按钮的后台，而是一个更加稳定、可验证、可生成、可复用的内容协议层。

What it is

Content Protocol定义作品、节点、人物、图片、文章、场地、规格与关系的结构。
File-based CMS基于 JSON / Markdown / manifests / scripts 的轻量内容中台。
Multi-site source layer同一份内容供 portfolio / collective / spaceport / newsroom / viewer 共同读取。
Agent-friendly适合被 intake / validation / generation / sync 等 skills 操作。

What it is not

Not a page builder不负责直接拖拽页面样式。
Not a single site repo不等同于 portfolio，也不等同于 SpacePort。
Not a visual admin dashboard第一阶段不做重型后台。
Not a replacement不替代 Collective / SpacePort / Newsroom / SceneForge，而是给它们提供内容源。

Object Model

核心内容对象

这套系统不是围绕“页面”建立的，而是围绕“对象”建立的。页面只是这些对象的不同视图。第一版建议固定 9 类对象。

person

艺术家、协作者、组织者。适合挂社交链接、角色、简介、头像、相关作品。

collective

团队、网络、组织。适合挂成员、项目、公开介绍、分工结构。

work

作品、系列、project line。是 portfolio 和 collective 的核心内容单位。

node

一次公开呈现：展览、演出、camp、工作坊、发布节点、合作节点。

asset

图片、海报、截图、视频、文档。用于 image wall、archive、作品页与研究页。

writing

评论、研究、策展文、文章、复盘。供 Newsroom 与作品页读取。

venue

场地与屏幕环境。以后可以支撑 dome / LED / multi-screen / gallery viewer。

spec

技术规格：分辨率、帧率、codec、container、Apple ProRes、硬件要求。

relation

对象之间的关系。它决定内容如何在网站、archive、timeline、viewer 中互相链接。

Workflow

内容如何进入网站系统

推荐工作流不是“直接改页面”，而是“先改内容，再生成页面可读数据”。这样后续加独立页、viewer、timeline tree、image wall 才不会不断出错。

1. 写入 source content在 `content/people/`、`content/works/`、`content/nodes/`、`content/specs/` 等目录中更新 JSON / Markdown。
2. validate schemas通过 validator 检查字段是否缺失、链接是否断裂、对象之间的关系是否成立。
3. generate site-specific data根据 site profiles 生成 portfolio / collective / newsroom / viewer 所需的数据层。
4. frontend consumes generated outputs前台站点只读取 `generated/` 结果，不直接把内容写死在组件里。
5. deploy public sites各前台站点继续保留自己的展示逻辑，但共享同一套协议层。
Use Cases

它能解决什么实际问题

这套系统不是为了更抽象，而是为了减少网站越来越复杂以后反复出错。下面是几个直接的用例。

Use case 01 — Personal portfolio

同一件作品同时出现在首页 Selected Works、Image Wall、作品详情页、创作谱系图、服务案例说明里，但不需要手改五份文案。

Use case 02 — Collective site

一个节点既能作为团队公开合作呈现出现，也能挂在个人成员履历下，还能进入 SpacePort 作为公共节点。

Use case 03 — Viewer / venue page

一个场地对象可以携带 dome 尺寸、分辨率、帧率、Apple ProRes、播放设备等参数，并自动挂到相关作品下。

Use case 04 — Newsroom linking

一篇评论既可以属于 Newsroom，也可以作为作品页里的 related writing，避免评论和作品信息彻底断开。

Onboarding

接入指南

第一阶段不要先做复杂后台。先让仓库、schema、内容目录和生成脚本成立，再逐步接到 portfolio 与其他前台站点。
